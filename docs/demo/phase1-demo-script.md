# 一期演示脚本

> **用途：** 用于内部评审、甲方阶段性演示和功能串讲。目标是在最短路径内展示“知识库管理 → 用户问答 → 问题反馈 → 管理员闭环处理”这一条完整主线。

---

## 1. 演示目标

本次演示重点不是展示底层模型能力，而是展示一期系统已经打通的业务闭环：

```text
管理员创建知识库
→ 上传文档
→ 授权用户
→ 用户进入问答页提问
→ 用户反馈答案无用
→ 系统生成知识缺口
→ 管理员回后台查看并处理知识缺口
```

建议把演示时间控制在 **5-10 分钟**。

---

## 2. 启动方式

在项目根目录下启动后端服务，例如：

```bash
cd backend
uvicorn app.main:app --reload
```

默认访问地址：

```text
http://127.0.0.1:8000
```

---

## 3. 默认账号

当前最小登录账号：

```text
用户名：admin
密码：admin
```

说明：

- 当前阶段主要用于功能验证和展示；
- 后续可接真实用户体系、系统管理员后台和组织权限。

---

## 4. 演示前准备

建议提前准备 1 个演示知识库和 1 份演示文档，或在演示时现场创建。

### 推荐演示知识库名称

```text
P368 产品知识库
```

### 推荐演示文档内容

可使用 `.txt` 文档，内容尽量短且稳定命中，例如：

```text
SOS 报警可以在设置中的报警设置里关闭。

MCSTARS 支持统一调度。
```

如果希望展示 `.docx` 也可以，但现场演示最稳的是 `.txt`。

---

## 5. 推荐演示顺序

## 第一步：登录进入管理台

访问：

```text
http://127.0.0.1:8000/login
```

使用：

```text
admin / admin
```

登录后进入 `/admin`。

**讲解要点：**

- 当前系统有管理员工作台；
- 管理员可在这里管理知识库、文档、权限和知识缺口。

---

## 第二步：创建知识库

在 `/admin` 页面顶部使用“创建知识库”表单：

- 名称：`P368 产品知识库`
- 描述：`P368 手册和 FAQ`
- 可见范围：`department`

点击：

```text
创建知识库
```

**预期结果：**

- 页面提示“知识库创建成功。”
- 左侧知识库列表刷新
- 新知识库自动被选中

**讲解要点：**

- 一期管理员可以直接在页面创建知识库；
- 创建者自动获得完整管理权限。

---

## 第三步：上传文档

在文档上传区选择准备好的演示文档，例如：

```text
p368-demo.txt
```

点击：

```text
上传文档
```

**预期结果：**

- 页面提示“上传成功。”
- 文档列表刷新
- 新文档自动选中
- 文档详情区显示：
  - 文件类型
  - 状态
  - 块数
  - 切片数

**讲解要点：**

- 一期支持文档上传与最小解析入库；
- 当前能看到解析状态和切片统计；
- 后续可以继续增强更多文件类型和解析能力。

---

## 第四步：授权用户（演示讲解型）

在权限编辑区输入用户 ID 并勾选权限。

当前页面是最小实现，展示时可说明：

- 一期已经具备按知识库、按用户授权的能力；
- 权限粒度包括：
  - 查看
  - 上传
  - 删除
  - 授权
- 目前页面输入的是用户 ID，后续会接更完整的用户选择器。

如果现场只做单账号演示，也可以把这一段作为“功能说明”而不是现场操作重点。

---

## 第五步：进入问答页

点击 `/admin` 页面的：

```text
进入问答页
```

进入 `/qa`。

**讲解要点：**

- 查询用户端和管理员端已经打通；
- 当前问答页支持知识库选择、提问、来源展示、历史会话和反馈。

---

## 第六步：选择知识库并提问

在 `/qa` 页面：

- 选择知识库：`P368 产品知识库`
- 输入问题：

```text
SOS 报警怎么关闭？
```

点击：

```text
提问
```

**预期结果：**

- 答案区显示命中答案；
- 来源区显示来源文档片段；
- 会话历史区出现当前会话；
- 历史消息区出现本轮问答；
- 后端已生成 `conversation_id` 和 `message_id`。

**讲解要点：**

- 当前问答已经不是一次性返回，而是会被记录成会话和消息；
- 用户可以回看历史会话，并在原会话继续追问。

---

## 第七步：提交“无用”反馈

在问答页反馈区输入：

```text
回答没有说清楚具体菜单路径
```

点击：

```text
无用
```

**预期结果：**

- 页面提示：

```text
已记录问题，管理员会处理。
```

- 后端生成一条 `open` 的知识缺口。

**讲解要点：**

- 一期已经具备问答反馈闭环；
- 无用反馈不会丢失，而是沉淀为待处理问题。

---

## 第八步：返回管理台查看知识缺口

点击 `/qa` 页面的：

```text
返回管理台
```

在 `/admin` 的知识缺口区查看刚才生成的问题。

**预期结果：**

- 能看到问题内容；
- 能看到反馈原因 / 文本；
- 可以点击：
  - `标记已处理`
  - `忽略`

**讲解要点：**

- 管理员已经可以看到真实用户问题；
- 一期已经把“用户问答反馈 → 管理员处理”闭环打通。

---

## 第九步：处理知识缺口

点击：

```text
标记已处理
```

或：

```text
忽略
```

**预期结果：**

- 页面提示“知识缺口状态已更新。”
- 当前列表刷新

**讲解要点：**

- 一期不仅能发现问题，还能在后台管理问题状态；
- 后续可继续扩展为更正式的运营看板与问题处理流程。

---

## 6. 演示时建议强调的价值点

建议在演示时明确强调以下 4 点：

### 1. 管理闭环已通

- 知识库创建
- 文档上传
- 权限授权
- 知识缺口处理

### 2. 查询闭环已通

- 用户可选择知识库问答
- 可查看来源
- 可查看历史会话

### 3. 反馈闭环已通

- 用户可反馈答案质量
- 无用反馈自动沉淀为知识缺口

### 4. 运营闭环已通

- 管理员能在后台看到知识缺口
- 可标记已处理 / 忽略

---

## 7. 常见演示风险与规避建议

### 风险 1：文档内容命不中

**建议：**

- 使用短小稳定的 `.txt` 演示文档；
- 问题文案尽量贴近文档原句。

### 风险 2：上传后看不到效果

**建议：**

- 上传后先展示文档详情区里的块数和切片数；
- 再进入问答页提问。

### 风险 3：权限演示复杂

**建议：**

- 如果现场只有一个账号，权限部分以“功能说明 + 页面展示”为主；
- 不必强行做多账号现场切换。

### 风险 4：展示时间过长

**建议：**

- 主线按“创建知识库 → 上传文档 → 问答 → 无用反馈 → 知识缺口处理”走；
- 不要现场展开过多技术实现细节。

---
