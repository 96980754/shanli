# 统一后台工作台前端优化设计

**日期：** 2026-07-14  
**方案：** B，统一后台工作台。  
**目标：** 在不引入前端框架和构建工具的前提下，将 `/login`、`/documents`、`/admin`、`/qa` 优化为统一、清晰、适合一期验收的企业后台界面。

---

## 1. 范围

本次优化包含：

1. 统一 4 个页面的深色后台视觉系统；
2. 优化 `/login` 登录注册页，增加验收账号提示；
3. 重点优化 `/documents` 权限文档工作台，使真实样本文件、分类权限、下载按钮更清晰；
4. 优化 `/admin` 管理台布局，使知识库、上传、文档、权限和规则区域更易操作；
5. 优化 `/qa` 问答页，使知识库、会话、提问、回答、来源和反馈区域更清楚；
6. 保留所有现有 DOM ID 和现有接口调用能力，避免破坏测试和既有 JS 行为。

本次不包含：

- React/Vue 或任何构建工具；
- 图标库依赖；
- 文档在线预览；
- 拖拽上传；
- 文档级权限；
- 组织架构同步。

---

## 2. 全局视觉系统

扩展 `backend/app/static/app.css`，形成一套可复用后台样式。

新增或增强类名：

- `.layout-dashboard`：页面级后台布局；
- `.sidebar`：左侧导航或分类栏；
- `.content-grid`：主要内容栅格；
- `.toolbar`：筛选与操作栏；
- `.data-card`：通用数据卡片；
- `.document-card`：文档卡片；
- `.metric-card`：统计卡片；
- `.status-badge--success`：已解析、可用；
- `.status-badge--warning`：仅下载、待处理；
- `.status-badge--danger`：无权限、错误；
- `.primary-action`：主操作按钮；
- `.secondary-action`：次操作按钮；
- `.danger-action`：危险操作按钮；
- `.field-list`：详情字段列表；
- `.split-panel`：左右分栏面板。

视觉原则：

- 背景保持深蓝灰；
- 主内容使用层级卡片；
- 可点击项有 hover 和 focus-visible；
- 下载、提问、保存等主操作必须醒目；
- 空状态必须说明原因和下一步；
- 响应式保持窄屏单列。


## 4. `/documents` 设计

这是本次重点页面。

### 4.1 布局

桌面端三栏：

```text
左栏：资料分类 / 知识库
中栏：文档列表和筛选
右栏：文档详情与下载
```

顶部：

- 系统名；
- 当前账号；
- 导航入口：文档库、问答页、管理台；
- 退出按钮。

### 4.2 左栏

展示用户可访问知识库：

- 知识库名称；
- 文档数量；
- 当前选中态；
- 无权限时显示“暂无可访问资料，请联系管理员”。

### 4.3 中栏

顶部工具栏：

- 搜索框（先做前端标题过滤）；
- 解析状态筛选；
- visibility 筛选；
- 产品筛选。

文档卡片展示：

- 文件名；
- 所属资料库；
- 文件类型；
- 产品；
- 解析状态；
- 下载状态。

状态文案：

- `parsed`：已解析，可用于问答；
- `stored_unsupported`：仅可下载，暂不支持内容解析；
- `pending`：等待处理。

### 4.4 右栏

详情区展示：

- 文件名；
- 原始文件名；
- 文件类型；
- 文件大小；
- 解析状态；
- 是否可问答；
- 是否可下载；
- 部门；
- 产品；
- visibility；
- 密级；
- block/chunk 数。

下载按钮：

- 可下载：主按钮“下载原文件”；
- 不可下载：禁用按钮，并显示原因；
- PPTX/XLSX：额外提示“此文件保留下载，不进入问答索引”。

### 4.5 管理员区

仅 `can_grant` 用户可见。保持现有授权能力，但样式改成两块：

- 用户权限；
- 知识视图规则。

## 5. `/admin` 设计

改为后台管理布局：

```text
左侧：知识库列表 + 创建知识库
中间：上传文档 + 文档列表
右侧：文档详情 + 删除操作
底部：权限管理 + 知识视图规则 + 知识缺口
```

优化点：

- 创建知识库区域放在清晰卡片中；
- 上传表单突出文件选择和元数据；
- 文档详情用字段列表展示；
- 删除按钮使用危险色；
- 权限与知识视图规则分组；
- 空状态提供操作提示。

要求：保留所有现有 ID，不重写 `admin.js` 的核心数据流。

---

## 6. `/qa` 设计

改为问答工作台：

```text
左侧：知识库选择 + 历史会话
中间：消息历史 + 提问框
右侧：来源文档 + 反馈
```

优化点：

- 当前知识库选择更明显；
- 历史会话卡片化；
- 用户问题和系统回答视觉区分；
- 来源文档卡片化；
- “有帮助”“没有帮助”按钮更清楚；
- 无知识库时显示权限说明。

## 8. 验收标准

1. `/login` 像正式系统入口，而不是裸表单；
2. `/documents` 能清楚看到 6 个资料分类和真实样本文件；
3. 下载按钮醒目、可用状态明确；
4. DOCX/PDF 与 PPTX/XLSX 状态区分清楚；
5. 不同账号权限差异可直观看到；
6. `/admin` 的上传、权限、规则区域更易操作；
7. `/qa` 的问答和来源展示更清楚；
8. 所有现有接口和测试继续通过。
