# 真实样本知识库演示与权限工作台设计

**日期：** 2026-07-13  
**目标：** 基于 `AI知识库_数据样本包.zip` 建立可重复的真实数据演示环境：全量导入 16 份样本，按权限分级验证文档浏览、下载与管理；同时将现有静态页面统一升级为深色专业运营工作台风格。

---

## 1. 范围

本阶段包含：

1. 显式运行的演示数据初始化命令；
2. 导入 ZIP 中全部 16 份真实样本文件；
3. 为每份文档写入预定义的分类、产品线、可见性、密级和标签；
4. DOCX/PDF 完成解析并进入问答索引；
5. PPTX/XLSX 保存原文件、可下载，但明确标识为暂不支持内容解析；
6. 创建 5 个演示用户和对应知识库权限、知识视图规则；
7. 将 `/documents` 升级为权限感知的文档与权限测试工作台；
8. 为登录、管理台、问答和文档工作台建立统一的深色专业运营工作台视觉系统；
9. 覆盖初始化、权限可见范围、不支持解析状态和前端壳的自动化测试；
10. 提供真实数据手工验收步骤。

本阶段不包含：

- PPTX/XLSX 的文本、表格或图片解析；
- 在应用启动时自动写入演示数据；
- 将 68 MB 样本解压后提交到仓库；
- 对 RAG 模型、向量库、OCR/VLM 的改造；
- 重构为 React 或引入 CSS/UI 构建工具；
- 多租户、角色系统重构或生产用户管理。

## 2. 真实样本导入

### 2.1 初始化命令

新增仅供开发/演示使用的 Python 模块命令：

```bash
cd backend
python -m scripts.seed_demo_data \
  --archive ../data/examples/AI知识库_数据样本包.zip \
  --database sqlite:///../data/demo.db \
  --reset
```

参数语义：

- `--archive`：样本 ZIP 的绝对或相对路径；必须存在并是可读取的 ZIP；
- `--database`：目标 SQLite URL；
- `--reset`：删除该演示数据库中本命令创建的知识库、用户、关联文档/解析数据和演示文件存储，然后重新创建；
- 无 `--reset` 时，若检测到同名演示数据已存在，命令不新增重复数据，输出如何使用 `--reset` 重建的说明并以非零状态退出。

命令仅在被显式调用时改写目标数据库和演示文件目录。应用启动不自动加载 ZIP，不写入用户，不改变现有部署的业务数据。

ZIP 在临时目录中解压；原始样本包和解压文件都不纳入 Git。导入后的原文件仍通过 `LocalFileStorageService` 保存为相对 `storage_key`，遵循应用现有的上传、下载和删除行为。

### 2.2 文档处理状态

| 文件格式 | 导入结果 | `Document.status` | 问答索引 | 下载 |
|---|---|---|---|---|
| DOCX、PDF | 保存并调用现有解析服务 | `parsed` | 写入 ContentBlock / DocumentChunk | 可用 |
| PPTX、XLSX | 保存原文件，不调用不支持的解析器 | `stored_unsupported` | 不写入 | 可用 |

`stored_unsupported` 表示成功保存而非失败。所有 API 和前端均需将其展示为：

```text
仅可下载（暂不支持内容解析）
```

列表和详情的数据库模式文档响应应稳定包含：

```json
{
  "parse_available": false,
  "parse_status_label": "仅可下载（暂不支持内容解析）",
  "download_available": true
}
```

`parse_available` 仅当状态为 `parsed` 时为 `true`。历史文档的缺失原文件仍按既有语义：`download_available=false`。

## 3. 演示身份与权限

初始化器创建一个 `MCSTARS 产品知识库` 和以下用户。所有账户的演示密码均为 `Demo123!`，仅用于本地演示数据库，不写入 HTML、JavaScript 或生产默认配置。

| 用户名 | 知识库权限 | 知识视图规则 | 验收目的 |
|---|---|---|---|
| `admin` | `can_view`、`can_upload`、`can_delete`、`can_grant` | 无；`can_grant` 自动绕过规则 | 可见全部 16 份并可管理授权 |
| `public_viewer` | `can_view` | visibility=`public`，密级 ≤ 1 | 仅验证公开资料 |
| `delivery_viewer` | `can_view` | visibility=`public` 或 `internal`，密级 ≤ 2，部门=`市场` 或 `交付` | 验证公开及交付资料 |
| `ops_viewer` | `can_view` | visibility=`internal` 或 `restricted`，密级 ≤ 3，部门=`运维` | 验证运维部门隔离 |
| `blank_user` | 无 | 无 | 验证初始不可访问，供管理员在界面上临时授权 |

知识视图规则的跨维度语义维持现有实现：不同非空维度为 AND，同维度多值为 OR。因此 `ops_viewer` 不可因 visibility 匹配而看到不属于“运维”部门的 restricted 文档。

演示账号快捷填充只能填写用户名；密码输入由用户在登录页填写。工作台显示“演示账号”说明及预期访问范围，但不泄露密码。

## 4. 前端信息架构和视觉系统

### 4.1 视觉方向

所有静态页面共用“深色专业运营工作台”视觉语言：

- 深蓝灰背景与分层表面色；
- 青蓝强调色，克制使用状态色区分成功、提醒、受限和错误；
- 高对比、可读的中英文字体栈；
- 统一的卡片、数据表格、标签、按钮、表单、空状态和反馈样式；
- 键盘焦点、禁用状态、状态文本不只依靠颜色区分；
- 桌面优先，窄屏下导航、三栏工作台和表格线性收敛。

新增 `backend/app/static/app.css` 为唯一共享样式入口。登录、管理台、问答和文档工作台均引入该文件；不引入前端框架或构建步骤。

### 4.2 `/documents` 权限演示工作台

页面采用桌面三栏布局：

1. **顶部导航**：产品标识、当前登录账户、权限标签、跳转问答页/管理台、退出；
2. **左栏**：当前可访问知识库、各状态文档计数、文档分类筛选；管理员额外获得“测试用户与权限”入口；
3. **中栏**：可筛选文档列表，每项包含名称、格式、分类/产品线、visibility、密级、解析状态和下载状态；
4. **右栏**：文档详情、完整元数据、解析统计和下载操作；管理员展开后可在同一页查看用户、修改知识库权限和配置知识视图规则。

筛选仅在客户端对已获授权的列表结果执行；筛选状态包含“全部”“已解析”“仅可下载”，并可按 visibility、文档类型和产品线进一步缩小范围。接口授权永远由后端执行，前端筛选不扩大文档可见性。

`stored_unsupported` 在清单和详情中均以“仅可下载”标签、解释文本和可用下载按钮呈现。仅当 `download_available=true` 时按钮可用。

管理员权限编辑复用既有 API：

```text
GET/PUT/DELETE /api/kb/{kb_id}/permissions/{user_id}
GET/PUT/DELETE /api/kb/{kb_id}/view-rules/{user_id}
```

非管理员不渲染管理控件；后端仍以 `can_grant` 作为唯一授权依据。

### 4.3 其他页面

- `/login`：改为统一品牌登录卡片、输入状态和演示用户名快捷填充；
- `/admin`：保留既有功能和 DOM/API 交互，只使用共享导航、区块和表单样式；
- `/qa`：保留既有功能和 DOM/API 交互，只使用共享导航、会话、问答和来源样式；
- `/documents` 是本阶段唯一新增或明显扩展业务交互的页面。

## 5. 服务边界与错误处理

- 演示导入器通过数据库服务、`LocalFileStorageService` 和 `IngestionService` 写入数据，不手工伪造 `Document`、chunk 或权限记录；
- 对 PPTX/XLSX，导入器不调用 `IngestionService.ingest_uploaded_document()`，直接设置 `stored_unsupported` 并提交，以避免生成无效 `ParseTask`；
- DOCX/PDF 解析异常属于真实导入错误：该文件保留原文件与文档记录，状态设为明确失败状态，初始化报告列出文件及原因，并让命令以非零状态退出；
- 导入前验证 ZIP 的精确文件清单，避免因错误样本包产生错误的权限验收结论；
- `--reset` 的清理必须只针对初始化器识别的演示知识库和演示用户，不得删除其他数据库记录；
- 下载继续使用文档访问服务校验；成功下载继续写审计日志。

## 6. 测试与验收

### 6.1 自动化测试

新增或扩展测试应验证：

1. 受控 ZIP fixture 中的 16 个样本映射为正确元数据；
2. 初始化命令创建知识库、5 个用户和正确的权限/知识视图规则；
3. 已解析格式创建内容块与 chunk，PPTX/XLSX 为 `stored_unsupported` 且没有 chunk；
4. 所有 16 份都有可用 `storage_key`，包括 PPTX/XLSX；
5. 重复运行无 `--reset` 不生成重复数据，带 `--reset` 后仍精确创建一套演示数据；
6. 五类用户的文档可见集合符合权限矩阵；管理员为 `blank_user` 授权和写规则后，后者的可见集合发生预期变化；
7. 列表/详情返回 `parse_available`、`parse_status_label` 和正确的下载状态；
8. 前端壳包含共享样式入口、工作台关键区域、状态标签及管理员权限操作钩子；
9. 现有 API、文件下载和审计回归测试继续通过。

### 6.2 手工真实数据验收

1. 运行初始化命令，核对报告显示 16 份原文件、13 份已解析、3 份仅可下载；
2. 使用 `admin` 和 `Demo123!` 登录，确认可见全部 16 份、3 份 PPTX/XLSX 可下载并显示“仅可下载”；
3. 分别使用 `public_viewer`、`delivery_viewer` 和 `ops_viewer` 登录，确认只显示相应规则允许的文档；
4. 使用 `blank_user` 登录，确认无可访问知识库；
5. 重新使用 `admin` 登录，在工作台授予 `blank_user` `can_view`，再设置知识视图规则；
6. 使用 `blank_user` 重新登录，确认可访问的文档集合与所设置规则相符；
7. 从任一有权账户下载 DOCX、PDF、PPTX 和 XLSX 各一份，核对浏览器下载名和文件内容；
8. 使用管理员或数据库检查下载审计存在 `download_document` 记录。

## 7. 成功标准

- 真实 ZIP 全量 16 份文件可通过一条显式命令导入；
- 任意样本文件均不因解析支持差异而失去下载能力；
- 不支持格式的“保存成功、不可解析”状态在 API 和前端中不被误判为失败；
- 权限矩阵可通过独立登录账户稳定复现；
- 管理员可在工作台为无权限用户完成授权与知识视图配置；
- 四个静态页面共享一致的深色专业运营工作台风格；
- 自动化测试和真实导入验收均有明确、可重复的执行路径。
